# Guia para Construção da Página de Vendas - Sistema Member Get Member

## Objetivo
Criar uma página de vendas atraente e convertida para apresentar o sistema de indicações AgentSummary aos associados, destacando benefícios, funcionalidades e facilidade de uso.

## Estrutura da Página

### 1. Hero Section
**Componente**: Hero com valor proposição principal

**Elementos**:
- Título impactante: "Multiplique Suas Vendas com Indicações"
- Subtítulo: "Benefícios exclusivos de Membros pra membros com AgentSummary"
- CTA principal: "Começar Agora" / "Solicitar Demonstração"
- Imagem/Ilustração: Representação visual do sistema de indicações
- Badge de confiança: "Usado por X associados" ou "Aumente vendas em até X%"

**Técnicas de Conversão**:
- Destaque para benefício principal (aumento de vendas)
- CTA acima da dobra
- Visual limpo e profissional

### 2. Seção de Benefícios para associados
**Componente**: Grid de cards com benefícios

**Benefícios a destacar**:
- Aumento de vendas através de indicações
- Redução de custos de aquisição de clientes
- Engajamento de clientes existentes
- Diferentes tipos de recompensas (cashback, descontos, pontos, prêmios)
- Gestão completa de campanhas
- Analytics e métricas em tempo real
- Sistema antifraude integrado
- Fácil integração com sistema existente

**Design**:
- Cards com ícones
- Título + descrição curta
- Visual consistente

### 3. Seção "Como Funciona"
**Componente**: Timeline ou steps horizontais

**Passos**:
1. Criar campanha de indicação
   - Configure recompensas e regras
   - Defina período de validade
   
2. Compartilhar código de indicação
   - Gere código único para cada cliente
   - Compartilhe via WhatsApp, email ou link
   
3. Cliente indica alguém
   - Indicado usa código no cadastro/checkout
   - Sistema valida automaticamente
   
4. Recompensas automáticas
   - Quando indicação é completada
   - Recompensas processadas automaticamente
   - Notificações enviadas

**Design**:
- Números ou ícones para cada passo
- Visual progressivo (esquerda para direita)
- Exemplos visuais quando possível

### 4. Seção de Tipos de Recompensas
**Componente**: Cards ou tabs com tipos de recompensas

**Tipos**:
- **Cashback**: Dinheiro direto na carteira virtual
- **Desconto**: Cupons para próximas compras
- **Pontos**: Acúmulo no programa de fidelidade
- **Prêmios Físicos**: Produtos do catálogo AgentSummary

**Para cada tipo**:
- Ícone representativo
- Descrição breve
- Exemplo de uso
- Benefícios específicos

### 5. Seção de Campanhas Ativas (Demo)
**Componente**: Lista de campanhas de exemplo

**Elementos**:
- Cards de campanha com:
  - Nome da campanha
  - Tipo (single-tier, multi-tier)
  - Recompensas oferecidas
  - Status (ativa)
  - Métricas básicas (opcional)
- Filtros (se houver múltiplas)
- Link para ver mais detalhes

**Nota**: Esta seção pode mostrar campanhas reais se houver API disponível, ou exemplos estáticos para demonstração.

### 6. Formulário de Cadastro/Interesse
**Componente**: Formulário de contato ou cadastro

**Campos**:
- Nome completo
- Email
- Telefone
- Nome da franquia
- Cidade/Estado
- Mensagem (opcional)
- Checkbox de aceite de termos

**Validações**:
- Todos os campos obrigatórios (exceto mensagem)
- Validação de email
- Validação de telefone brasileiro
- Mensagem de sucesso após envio

**Integração**:
- Enviar para endpoint de leads ou contato
- Integrar com sistema de notificações
- Mostrar feedback visual (loading, sucesso, erro)

### 7. Depoimentos e Social Proof
**Componente**: Carrossel ou grid de depoimentos

**Elementos**:
- Foto do franqueado (ou avatar)
- Nome e franquia
- Depoimento/texto
- Métricas alcançadas (opcional)
- Rating/estrelas (se aplicável)

**Exemplos de depoimentos**:
- "Aumentei minhas vendas em 30% usando indicações"
- "Sistema muito fácil de usar, recomendo!"
- "Meus clientes adoram ganhar cashback"

### 8. FAQ (Perguntas Frequentes)
**Componente**: Accordion ou seção expansível

**Perguntas sugeridas**:
- Como funciona o sistema de indicações?
- Quais tipos de recompensas posso oferecer?
- Quanto custa usar o sistema?
- Como configuro uma campanha?
- Como são processadas as recompensas?
- Há limite de indicações?
- Como funciona o sistema antifraude?
- Posso personalizar as campanhas?
- Como vejo os resultados das minhas campanhas?
- Preciso de conhecimento técnico para usar?

**Design**:
- Accordion expansível
- Busca de perguntas (opcional)
- Categorias (opcional)

### 9. CTA Final
**Componente**: Seção de chamada para ação

**Elementos**:
- Título: "Pronto para aumentar suas vendas?"
- Subtítulo: "Comece hoje mesmo e veja resultados em poucos dias"
- Botão principal: "Começar Agora"
- Botão secundário: "Falar com Vendas"
- Links para: Demo, Documentação, Suporte

### 10. Footer
**Componente**: Rodapé padrão

**Elementos**:
- Links importantes
- Contatos
- Redes sociais
- Termos de uso
- Política de privacidade

## Componentes Necessários

### Componentes Base (Reutilizáveis)
1. **Button**: Botões com variantes (primary, secondary, outline)
2. **Card**: Cards para campanhas, benefícios, etc.
3. **Badge**: Badges para status, tipos, etc.
4. **Input**: Inputs de formulário
5. **Textarea**: Textarea para mensagens
6. **Accordion**: Para FAQ
7. **Carousel**: Para depoimentos
8. **Timeline**: Para seção "Como Funciona"

### Componentes Específicos
1. **CampaignCard**: Card de campanha de indicação
2. **RewardBadge**: Badge mostrando tipo de recompensa
3. **BenefitCard**: Card de benefício
4. **StepCard**: Card de passo do processo
5. **TestimonialCard**: Card de depoimento
6. **ContactForm**: Formulário de contato

## Integração com Backend

### Endpoints Necessários
1. **GET /referral-campaigns**: Listar campanhas ativas (para seção de campanhas)
2. **POST /leads** ou **POST /contact**: Enviar formulário de interesse
3. **GET /referral-campaigns/:id**: Detalhes de campanha específica

### Hooks React Query Sugeridos
```typescript
// hooks/useReferralCampaigns.ts
export const useActiveCampaigns = () => {
  return useQuery({
    queryKey: ['referral-campaigns', 'active'],
    queryFn: () => referralService.getActiveCampaigns(),
  });
};

// hooks/useContactForm.ts
export const useContactForm = () => {
  return useMutation({
    mutationFn: (data: ContactFormData) => contactService.submit(data),
  });
};
```

## Design e UX

### Cores e Estilo
- Seguir design system existente do AgentSummary
- Usar cores da marca
- Tipografia clara e legível
- Espaçamento adequado

### Responsividade
- Mobile-first approach
- Breakpoints: mobile, tablet, desktop
- Testar em diferentes tamanhos de tela

### Performance
- Lazy loading de imagens
- Code splitting para componentes pesados
- Otimização de imagens
- Cache de dados da API

### Acessibilidade
- Contraste adequado
- Navegação por teclado
- ARIA labels onde necessário
- Textos alternativos em imagens

## CTAs e Conversão

### Estratégia de CTAs
1. **Hero**: CTA principal acima da dobra
2. **Benefícios**: CTAs secundários após cada seção importante
3. **Final**: CTA forte antes do footer

### Otimização de Conversão
- Formulário simples e rápido
- Reduzir fricção (menos campos obrigatórios)
- Mostrar valor antes de pedir cadastro
- Social proof visível
- Garantias ou trial gratuito (se aplicável)

## Métricas a Rastrear

### Analytics
- Visualizações da página
- Cliques em CTAs
- Submissões de formulário
- Tempo na página
- Scroll depth
- Taxa de conversão

### Eventos Customizados
- `referral_sales_page_viewed`
- `cta_clicked` (com tipo)
- `form_started`
- `form_submitted`
- `campaign_card_clicked`
- `faq_expanded`
